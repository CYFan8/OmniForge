# 文案与交互话术模板

## 核心原则

- 清晰 > 准确 > 简洁（顺序不可颠倒）
- 说人话：优先用户能立即理解的语言，而非技术术语
- 一致：整个产品的术语、语气、标点符号保持一致
- 可扫读：用户扫一眼就能抓住关键信息

---

## 一、语气矩阵

| 场景 | 语气 | 例句 |
|------|------|------|
| 操作成功 | 轻松肯定 | 「已保存」vs「操作成功」 |
| 操作失败 | 有温度不推卸 | 「出了点问题，我们正在修」vs「服务器返回500错误」 |
| 确认操作 | 礼貌明确 | 「你确定要删除这份文档吗？删除后无法恢复。」 |
| 引导新手 | 友好耐心 | 「我们来一步步设置你的项目——先取个名字吧」 |
| 提示信息 | 中性简洁 | 「密码需要至少8个字符」 |
| 表单标签 | 动词开头 | 「输入邮箱」「设置密码」 |
| 错误提示 | 指出问题+解决方案 | 「邮箱格式不对，试试 user@example.com」 |

## 二、中英文规范

- 中英文混排时，中英文之间加半角空格（中文「使用 API 接口」）
- 中文使用「」作为引用标记
- 数字与单位之间加空格（「100 MB」「3 天前」）
- 英文专业术语保持原文不翻译（API, PDF, URL）
- 中文引号使用「」而非 ""

## 三、场景模板

### 3.1 空态（Empty State）

`
[图标/插画]
[主文案：发生了什么]
[副文案：为什么 + 用户可以做什么]
[行动按钮]
`

**例**：
`
📭（空收件箱图标）
还没有消息
当你收到团队消息时，会显示在这里
邀请队友开始协作 →
`

### 3.2 错误状态（Error State）

`
[问题描述] → [原因（可选）] → [解决方案（具体步骤）] → [行动按钮]

// 好
「无法加载这个文件。可能是文件已损坏或格式不支持。试试重新上传？」
// 不好
「发生未知错误，请联系管理员（错误码: 0x8A3F002B）」
`

### 3.3 确认对话框

`
[标题：决定性的问题]
[内容：后果 + 可逆性说明]
[主按钮：动词明确] [次按钮：取消/返回]
`

**例**：
`
删除这个项目？
所有数据将被永久删除，无法恢复。
[删除项目] [取消]
`

### 3.4 表单引导

`
[标签：动词/名词明确]
[占位符：示例值]
[帮助文字：可选格式提示]
[错误：具体修改建议]

例：
邮箱地址
user@example.com
[验证通过 ✓]
`

## 四、消息文案结构

### 通知消息

`
[对象] + [动作] + [结果/影响]
- 「文档「数据分析报告」已生成」
- 「你的分享链接已过期，重新生成 →」
`

### 进度消息

`
[当前步骤] + [总步骤] + [预计剩余时间（可选）]
- 「正在上传：3/5（约 2 分钟）」
- 「正在导出，完成后会通知你」
`

### 系统消息

`
[客观描述] + [可采取措施]
- 「你的会话已超时，需要重新登录」
- 「检测到新版本，刷新页面以更新」
`

## 五、常用 UI 元素文案速查

| 元素 | 推荐文案 | 避免 |
|------|---------|------|
| 保存按钮 | 「保存」 | 「提交」「确定」 |
| 取消按钮 | 「取消」「返回」 | 「上一步」「放弃」 |
| 删除按钮 | 「删除」+ 确认弹窗 | 直接删除 |
| 搜索框 | 「搜索项目/文件/联系人…」 | 「请输入关键词」 |
| 空列表 | 「还没有 XX」 | 「暂无数据」 |
| 加载中 | 具体内容「正在加载…」 | 「Loading…」 |
| 完成提示 | 「已完成」 | 「执行完毕」「Success」 |
| 权限不足 | 「你没有查看此内容的权限，联系管理员申请」 | 「403 Forbidden」 |
