# Code Engineer Loop - 生成 + 自检 + 完善 + 学习

## 核心哲学

> 不要只写代码，要为每次写代码积累经验。

本 Skill 将每次代码任务拆为四个阶段，形成一个可重复的闭环：

```
  +-----------------------------------------------------------+
  |                                                           |
  |   1. 生成代码 (Generate)                                   |
  |      |                                                    |
  |      v                                                    |
  |   2. 自检 (Check) ---- 不通过 ----> 3. 自我完善            |
  |      | 通过                      ^         (Improve)      |
  |      v                           +------------------------+
  |   4. 自我学习 (Learn)                                      |
  |      |                                                    |
  |      v                                                    |
  |   回到 1 或结束                                            |
  |                                                           |
  +-----------------------------------------------------------+
```

---

## 阶段一：生成代码 (Generate)

### 执行流程

1. 理解需求：先复述需求确认理解正确
2. 设计方案：输出简要方案(架构/接口/数据流)，请用户确认后再编码
3. 增量实现：一次只实现一个模块，完成后即进入自检阶段

### 输出规范

- 每个函数/类必须有类型注解(Python)或 TypeScript 类型
- 每个文件头部有模块职责注释
- 关键逻辑行有行内注释
- 生成配套的单元测试(覆盖率目标大于 80%)

### 依赖声明

```python
# requirements.txt 或 package.json 必须标明新增依赖及版本
```

## 阶段二：自检 (Check)

每次生成代码后，必须先自检再交付。按以下清单逐项检查：

### [P0] 编译/语法检查

- [ ] 代码无语法错误(IDE 或编译器验证)
- [ ] 类型检查通过(mypy / TypeScript strict 模式)
- [ ] 导入/依赖无缺失

### [P1] 正确性检查

- [ ] 单元测试全部通过(必跑)
- [ ] 边界条件已覆盖(空值、极值、异常输入)
- [ ] 核心逻辑可以用一句话说清(说不清说明设计有问题)

### [P2] 质量检查

- [ ] 无硬编码(魔法数字、字符串常量已提取)
- [ ] 错误处理完备(try/except 或 Result 类型)
- [ ] 日志/调试输出已移除或转为正式日志
- [ ] 函数长度小于 50 行，单个文件小于 500 行
- [ ] 命名符合项目规范

### [P3] 安全检查

- [ ] 无 SQL 注入风险(使用参数化查询)
- [ ] 无命令注入风险(避免 shell=True)
- [ ] 敏感信息(密码/token/key)不在代码中硬编码

### 自检结果输出格式

```markdown
## 自检报告

| 等级 | 状态 | 项数 |
|------|------|------|
| P0   | 通过/失败 | 3/3  |
| P1   | 通过/失败 | 3/3  |
| P2   | 通过/失败 | 6/6  |
| P3   | 通过/失败 | 3/3  |

结论: 全部通过 / 存在 N 项待修复
```

- 若全部通过 -> 进入阶段四(自我学习)
- 若存在未通过项 -> 进入阶段三(自我完善)

---

## 阶段三：自我完善 (Improve)

针对自检发现的问题，按优先级逐一修复：

### 修复优先级

```
P0 编译错误 --> P1 正确性 --> P2 质量 --> P3 安全
  最高            高           中          高
```

### 修复流程

1. 定位：找到问题代码位置
2. 诊断：分析根因(不要只修表象)
3. 修复：实施修改
4. 验证：重新跑对应检查项确认修复成功
5. 回归：确保修复没有引入新问题

### 最大尝试轮次

- 每个问题最多尝试 3 次修复
- 3 次仍未修复 -> 记录阻塞原因，向用户报告并请求帮助

### 修复后

- 回到 阶段二 重新执行完整自检
- 最多循环 5 轮，超过则向用户报告并请求人工介入

---

## 阶段四：自我学习 (Learn)

这是本 Skill 区别于普通代码生成的核心能力。每次任务完成后，将经验沉淀下来。

### 学习产出物

#### 4.1 更新 CLAUDE.md

将本次任务中发现的**项目级知识**追加到项目根目录的 CLAUDE.md：

```markdown
## 经验记录 (由 code-engineer-loop 自动沉淀)

### YYYY-MM-DD: [任务简述]
- 问题: [遇到的坑/踩过的雷]
- 原因: [根因分析]
- 方案: [最终采用的解决方案]
- 下次建议: [如果有类似场景，直接怎么做]
```

#### 4.2 沉淀 Skills

当发现**可跨项目复用的通用知识**时，自动在 .claude/skills/ 下创建新的 Skill 文件：

```markdown
---
name: short-descriptive-name
description: 一句话说明什么场景下使用这个 Skill
---

# Skill 名称

## 核心知识点
- [提炼 3-5 条关键经验]

## 常见坑
- [已知的陷阱和避免方法]

## 可复用模板

```
[代码片段或配置模板]
```
```

#### 4.3 记录到学习日志

在 .claude/learn.log 中追加一行(如果文件不存在则创建)：

```
YYYY-MM-DD HH:MM | <任务类型> | <文件/模块> | <关键经验一句话>
```

### 什么值得学

| 应沉淀 | 不该沉淀 |
|--------|---------|
| 踩了 30 分钟以上的 Bug | 显而易见的语法修正 |
| 非显而易见的配置坑 | 拼写错误 |
| 项目特有的约定和模式 | 标准 API 使用方式 |
| 跨模块的架构决策 | 单行代码调整 |
| 性能优化经验 | 格式化调整 |

## 附加要求

### 当用户说"直接写"时

如果用户说"直接写代码"或"不用检查"，仍然执行完整的四阶段流程，但可以加快自检速度(只跑 P0 + P1)。

### 当任务特别简单时

仅修改一行配置、重命名变量等原子操作：
- 跳过阶段三(无需完善)
- 阶段二只检查 P0
- 阶段四只记录 learn.log(不更新 CLAUDE.md)

### 上下文不够时

如果上下文窗口接近上限，优先保留：
1. 阶段二的检查清单(未完成项)
2. 阶段四待写入的学习内容
3. 当前正在修改的文件

## 数学建模方法代码库

使用数学建模方法时，优先引用 `math_modeling_toolkit.py` 中的实现，避免重复造轮子。

### 可用模块速查

| 模块 | 函数 | 用途 |
|------|------|------|
| 图论 | `bfs_shortest_path`, `bfs_path`, `dijkstra`, `floyd_warshall` | 最短路径/全源距离 |
| DP | `knapsack_01`, `edit_distance` | 背包问题/序列比对 |
| 蒙特卡洛 | `monte_carlo_integrate`, `monte_carlo_simulate`, `latin_hypercube_sample` | 积分/模拟/降方差 |
| 马尔可夫 | `MarkovChain` 类 | 状态转移/稳态分布 |
| 回归 | `linear_regression`, `multiple_linear_regression` | 一元/多元线性回归 |
| 灰色预测 | `gm11_predict` | GM(1,1)小样本预测 |
| 评价 | `ahp_weight`, `topsis`, `entropy_weight` | AHP/TOPSIS/熵权法 |
| 博弈 | `nash_equilibrium_2x2` | 2人纳什均衡 |
| 时间序列 | `exponential_smoothing` | 指数平滑预测 |
| 聚类 | `kmeans` | K-means聚类 |

### 使用方法

```python
from math_modeling_toolkit import *

# 示例：评价问题
data = [[8,7,2], [7,8,4], [6,9,6]]  # 3方案×3指标
weights = entropy_weight(data)        # 熵权法求权重
result = topsis(data, weights, [True, True, False])  # TOPSIS评价
print(f'最优方案: {result["ranking"][0]}, 得分: {result["scores"]}')
```

### 方法选择检查

编写建模代码前，先确认：
1. 问题类型（优化/预测/评价/随机/博弈）→ 选择对应模块
2. 数据特征（样本量/线性/缺失）→ 调整方法或参数
3. 避免"锤子思维"——不要所有问题都用同一个方法

## 基于Pillow的题目原图修改工具

用于数学建模中直接在题目所给地图/示意图上绘制路线、标注节点、添加文字说明，替代重绘示意图。

### 基础框架

```python
from PIL import Image, ImageDraw, ImageFont
import math

img = Image.open("原图.png")
draw = ImageDraw.Draw(img)
width, height = img.size  # 原点(0,0)在左上角,x右,y下
# ... 修改操作 ...
img.save("修改后.png")
```

### 颜色规则：RGB(0-255)，PNG透明图RGBA四通道(alpha:0全透明-255不透明)

### 绘图操作

```python
# 直线: 起点→终点, 颜色, 线宽
draw.line(xy=[(100,200),(500,200)], fill=(255,0,0), width=3)

# 矩形: 空心(outline)或填充(fill)
draw.rectangle(xy=[(50,50),(300,200)], outline=(0,0,255), width=2)
draw.rectangle(xy=[(50,250),(300,400)], fill=(0,255,0,128))  # 半透明填充

# 圆/椭圆: 外接矩形坐标
draw.ellipse(xy=[(100,100),(300,300)], outline=(255,0,0), width=2, fill=(255,255,0))

# 多边形: 按坐标依次连接
draw.polygon(xy=[(200,100),(100,300),(300,300)], fill=(128,128,128), outline=(0,0,0))
```

### 颜色修改

```python
# 单像素改色
img.putpixel((150,150), (255,0,0))

# 全局颜色替换(遍历像素)
img = img.convert("RGB"); pixels = img.load()
for x in range(width):
    for y in range(height):
        if pixels[x,y] == (255,255,255): pixels[x,y] = (240,240,240)

# 容差替换(处理渐变/噪点)
def color_dist(c1,c2): return math.sqrt(sum((a-b)**2 for a,b in zip(c1,c2)))
tolerance = 30
for x in range(width):
    for y in range(height):
        if color_dist(pixels[x,y][:3], (255,0,0)) <= tolerance:
            pixels[x,y] = (0,0,255)
```

### 文字与箭头标注

```python
# 文字(需系统字体)
font = ImageFont.truetype("C:/Windows/Fonts/msyh.ttc", size=24)
draw.text(xy=(100,50), text="起点S", fill=(0,0,0), font=font)

# 带箭头标注线
def draw_arrow(draw, start, end, color=(255,0,0), width=2, arrow_size=10):
    draw.line([start, end], fill=color, width=width)
    angle = math.atan2(end[1]-start[1], end[0]-start[0])
    p1 = (end[0]-arrow_size*math.cos(angle-math.pi/6), end[1]-arrow_size*math.sin(angle-math.pi/6))
    p2 = (end[0]-arrow_size*math.cos(angle+math.pi/6), end[1]-arrow_size*math.sin(angle+math.pi/6))
    draw.polygon([end, p1, p2], fill=color)

draw_arrow(draw, (100,100), (300,300), color=(255,0,0), width=2)
```

### 注意事项
- PNG透明图用`img.convert("RGBA")`统一格式
