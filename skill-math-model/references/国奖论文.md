## 一、核心理念

国奖论文 = 优秀的摘要 × 合理的结果 × 漂亮的论文

评审标准: 假设的合理性 → 建模的创造性 → 结果的正确性 → 文字表述的清晰程度

**核心避坑原则**：
- 「能用简单模型解决的，绝对不上复杂模型」
- 「建模不是列公式，而是把真实问题翻译成数学语言」
- 「每一个参数有来源，每一个假设有依据，每一个结论有验证」

## 二、论文标准结构（8段式）

```
1. 摘要（最关键，评阅第一印象，≤1页）
2. 问题重述（用自己的话，严禁照抄）
3. 问题分析（附技术路线流程图）
4. 模型假设与符号说明（三线表）
5. 模型的建立与求解（核心，占50%篇幅）
6. 结果分析与检验（灵敏度+模型对比+误差分析）
7. 模型评价与推广（优点3条+缺点诚实）
8. 参考文献（5篇以上）
附录: 完整可运行源代码（必需）
```

## 三、问题类型识别与模型选择（核心新增）

### 3.1 三步决策法

**第一步：定义问题目标类型**

| 目标类型 | 问题特征 | 首选模型 | 备选模型 |
|----------|----------|---------|---------|
| **预测** | 推断未来数值 | ARIMA | 回归、神经网络、灰色预测 |
| **优化** | 有限资源求最优解 | 线性/整数规划 | DP、遗传算法、粒子群 |
| **分类** | 分成不同群体 | SVM/随机森林 | K-means、决策树 |
| **评价** | 多方案打分排序 | TOPSIS+熵权法 | AHP、模糊综合评价 |
| **随机模拟** | 含不确定性的系统 | 蒙特卡洛 | MCMC、排队论 |
| **博弈** | 多方策略交互 | 纳什均衡 | 合作博弈、演化博弈 |

**第二步：审视数据特征**

| 数据特征 | 选择策略 |
|----------|---------|
| 样本<100条 | 别用神经网络，用SVM/随机森林/回归 |
| 样本100~10000条 | ARIMA、XGBoost、岭回归 |
| 样本>10000条 | LSTM、深度学习 |
| 线性关系 | 线性回归、线性规划 |
| 非线性关系 | 多项式回归、核方法、神经网络 |
| 时序周期性 | ARIMA/指数平滑（别用线性回归） |
| 共线性严重(VIF>10) | 岭回归/LASSO（别用普通OLS） |
| 数据缺失严重 | 灰色预测GM(1,1)、灰色关联 |
| 小样本贫信息 | 灰色系统理论（≤5组数据用德尔菲法） |

**第三步：用交叉验证诊断模型**

```
训练误差+测试误差都高→ 欠拟合→ 换更复杂的模型
训练误差低+测试误差高→ 过拟合→ 加正则化/增加数据
训练误差+测试误差都低→ 模型合适 ✓
```

### 3.2 经典错误搭配速查（避坑！）

| 错误做法 | 后果 | 正确修正 |
|----------|------|---------|
| 线性回归拟合非线性数据 | 预测完全失真 | 多项式回归/核方法 |
| GM(1,1)预测剧烈波动数据 | 拟合失效 | ≥10组用ARIMA，<5组用德尔菲 |
| AHP的CR>0.1仍使用 | 权重无意义 | 熵权法/CRITIC修正+TOPSIS |
| BP神经网络样本<100 | 严重过拟合 | SVM/随机森林+L2正则化 |
| 遗传算法解实时问题 | 收敛太慢 | Q-learning/强化学习 |
| 模拟退火解>50维问题 | 搜索低效 | 粒子群PSO |
| 模糊评价隶属度凭感觉 | 结果无依据 | FCM聚类确定隶属函数 |
| 评价模型不检验信效度 | 结论不可靠 | Cronbach alpha+KMO检验 |

### 3.3 建模质量保障四关卡

1. **参数估计**：用最小二乘/最大似然估计，做置信区间
2. **模型验证**：留出测试集 + 残差分析 + 灵敏度分析(±20%)
3. **假设评估**：每个假设说明依据，评估对结果的影响
4. **结果可视化**：图表标题即结论，误差棒必画，导出≥300dpi

## 四、优化类方法详表

| 方法 | 适用场景 | 关键词 | 前提条件 | 典型工具 |
|------|---------|--------|---------|---------|
| 线性规划 | 目标+约束均为线性 | 单纯形法 | 连续性、比例性 | Lingo, MATLAB linprog |
| 整数规划 | 决策变量为整数 | 分支定界、0-1规划 | 离散变量 | Lingo, Python pulp |

| 粒子群优化 | 高维连续优化 | 速度+位置更新 | >50维 | Python pyswarms |

## 五、预测与评价类方法详表

### 5.1 预测方法选择

| 数据量 | 时序特征 | 推荐方法 | 注意事项 |
|--------|---------|---------|---------|
| <100 | 无明显趋势 | 移动平均、指数平滑 | 短期预测 |
| 100-1000 | 有明显趋势 | ARIMA | 需平稳性检验 |
| 100-1000 | 季节性+趋势 | SARIMA | 需识别周期 |
| 很小(<30) | 指数型增长 | 灰色预测GM(1,1) | 发展系数|a|<2 |
| >10000 | 非线性复杂 | LSTM/XGBoost | GPU加速 |
| 多变量 | 有影响因素 | 多元回归、SVM | VIF检验 |

### 5.2 评价方法选择

| 场景 | 推荐方法 | 权重确定 |
|------|---------|---------|
| 数据充足、客观排序 | TOPSIS（首选）| 熵权法 |
| 需要融入经验 | AHP | 判断矩阵(CR<0.1) |
| 指标模糊难量化 | 模糊综合评价 | AHP辅助 |
| 小样本贫信息 | 灰色关联分析 | 客观赋权 |
| 高标准综合评价 | AHP+熵权法+TOPSIS | 主客观组合 |
| 对离群值不敏感 | 秩和比法(RSR) | 无需赋权 |

## 六、随机与统计类方法

### 6.1 蒙特卡洛模拟

- **适用**：系统含随机变量，需估计统计特征
- **前提**：能定义随机变量的概率分布，计算资源足够（N≥1000次）
- **关键**：大数定律收敛 O(1/sqrt(N))，需拉丁超立方抽样降方差
- **典型应用**：项目风险分析、期权定价、随机过程模拟

### 6.2 马尔可夫链

- **适用**：状态转移有"无记忆性"的系统（下一状态仅取决于当前状态）
- **前提**：能定义状态空间和转移概率矩阵，状态数不宜过大
- **关键**：验证遍历性和稳态分布的存在性
- **典型应用**：天气预测、用户行为建模、排队系统

### 6.3 排队论

- **适用**：服务系统（顾客到达+排队+服务+离开）
- **前提**：到达间隔和服务时间服从已知分布（常见：泊松到达/指数服务）
- **关键模型**：M/M/1、M/M/c、M/G/1
- **典型应用**：银行柜台优化、呼叫中心配置、交通流量分析

### 6.4 MCMC（马尔可夫链蒙特卡洛）

- **适用**：高维复杂分布的采样，贝叶斯后验估计
- **前提**：需定义目标分布（可差常数倍），算法收敛需诊断
- **典型应用**：贝叶斯统计推断、参数不确定性量化

## 七、符号书写规范

### 7.0 公式输出强制规范（最高优先级，覆盖所有冲突）

1. **所有数学符号必须使用 LaTeX 语法**，禁止以任何理由使用纯文本模拟。错误：`Vpi(s) = SUMa pi(a|s) [r + gamma Vpi(s')]`。正确：`$V^{\pi}(s) = \sum_a \pi(a|s) \sum_{s'} P(s'|s,a) [r(s,a,s') + \gamma V^{\pi}(s')]$`。

2. **希腊字母必须使用反斜杠命令**：错误：`alpha, beta, gamma, Delta, Sigma, pi`。正确：`\alpha, \beta, \gamma, \Delta, \Sigma, \pi`。

3. **上下标必须使用标准语法**：错误：`Vpi, w_i+1, c_w^(k)`。正确：`V^{\pi}, w_{i+1}, c_w^{(k)}`。

4. **求和/积分/极限必须使用标准命令**：错误：`SUM, INT, LIM`。正确：`\sum, \int, \lim`。

5. **分数必须使用 \frac**：错误：`(a+b)/(c+d)`。正确：`\frac{a+b}{c+d}`。

6. **行内公式用 `$...$`，独立公式用 `$$...$$` 或 `\begin{equation}...\end{equation}`**。

7. **如果用户提供的公式是截图或纯文本模拟，必须先将其转换为标准 LaTeX，再输出。** 违反以上任何一条的输出不合格。

### 7.1 DOCX 兼容写法（python-docx生成）

| 写法 | 含义 | 示例 |
|------|------|------|
| `x_i` | 下标 | `w_i = alpha × x_i + beta` |
| `x^2` | 上标 | `R^2 = 0.9734` |
| `x_{ij}` | 多字符下标 | `w_{ij}` 表示区域i到j的权重 |
| `alpha, beta, sigma` | 希腊字母 | `alpha = 0.05` |
| `>=, <=, !=, ->` | 关系符 | `x_i >= 0` |
| 真下标 | 用 `font.subscript = True` | python-docx渲染 |

### 7.2 禁止写法

| 禁止 | 替代 |
|------|------|
| `\( \)` `$$ $$` | 去掉定界符直接写 |
| `\\alpha` LaTeX命令 | 写 `alpha` |
| 截图公式 | 用公式编辑器或代码生成 |

### 7.3 公式排版规则

**LaTeX方案（国奖论文首选）**：用`equation`环境自动居中+连续编号；长公式用`aligned`嵌套等号对齐，编号在末行右侧；`\dot{\theta}`表示一阶导数，`\ln`正体函数名，`\left(\right)`自适应括号。

**Word/DOCX方案（普及方案）**：
- 快捷键`Alt+=`调出内置公式编辑器，直接输入LaTeX语法自动转换
- **居中公式+右编号**：最稳定方法是1行2列或1行3列无边框表格法——左/中列放公式(居中)，右列放编号(右对齐)，整表设无框线
- MathType/AxMath插件支持可视化输入、一键插入带编号公式、批量更新编号
- Mathpix截图即可生成LaTeX代码或Word公式；Word墨迹公式支持手写识别

**符号规范**：物理变量斜体（如r_0, theta_0, v_0）；微分d、函数名(ln,sin)、单位、常量用正体；矢量矩阵粗体；全文符号格式统一。

**编号规范**：全文连续编号(1)(2)或按章节编号(2-1)(2-2)，全程统一；编号右对齐，与公式同一水平中线，圆括号包裹。

**长公式规范**：换行处等号对齐，运算符前后保留统一空格；多行公式仅保留一个编号在末行右侧。

### 7.5 数理推导规范（强制）

**推导前四要素**：研究对象 + 求解框架（坐标系/数学方法）+ 前提条件（所有基础定义/已知公式/参数数值与单位/变量约束）+ 推导目标（最终结果形式）。

**公式排版强制**：核心推导公式单独居中，按出现顺序连续编号(1)(2)(3)…，编号右对齐。长公式分段对齐，编号在末行右侧。数学符号标准语法，物理变量斜体，微分符号d正体；一阶导数用变量上方加圆点，二阶用双点。

**符号释义强制**：首次出现的物理量在公式后紧跟"其中，XX为XX"句式解释；带数值常量同时标注数值与单位（如d=0.55m为螺距）；变量约束条件（如θ_0<0）单独明确说明，不得省略。

**行文强制**：开篇1-2句正式学术陈述引入思路。推导正文用自然段落，禁止列表/分点序号。全程正式学术书面语，无口语化、无额外解读、无步骤标题。

**推导逻辑强制**：每步公式变形前配中文过渡说明（如"根据定义可得""分离变量后""两边同时积分可得"），不得无说明直接抛公式。不跳过关键变形步骤，所有代数变形/积分/求导完整呈现。从基础定义逐步推导至目标结果：引入→基础定义式→中间推导式→最终结果式。

## 十二、论文结构红线检查（逐条必过）

### 12.1 摘要页
- 含问题概述、方法概述、每问关键数值结果（带单位）、结论、创新点、4-6个关键词
- 每问必须给具体数字，禁止空话；必须含对比基线+灵敏度结论

### 12.2 问题重述
- 用自己的语言重新组织（非照抄），提炼为数学问题：给定XX，求XX，使XX最大化

### 12.3 模型假设
- 条数5-8条适中，每条在建模节有对应引用，关键假设解释合理性

### 12.4 符号说明
- 三线表四列：符号/含义/单位/取值范围；首次出现时正文解释
- 斜体变量，正体常数/函数名

### 12.5 模型建立与求解
- 每问独立成节("问题→模型→求解→结果"三段式)
- 建模前说明方法选择理由；公式为可编辑对象（非截图）
- 编号用制表位右对齐；多行公式等号对齐；求解不跳步
- 算法用伪代码/流程图描述并标注复杂度
- 每张图表有编号标题且在正文显式引用

### 12.6 模型检验
- 至少一个关键参数灵敏度分析（给出扰动后数值）
- 误差分析（模型误差/数据误差/舍入误差）
- 与真实数据或其他方法对比；极端条件测试

### 12.7 模型评价与推广
- 优点不夸大，缺点不回避，改进方向具体
- 推广到其他场景的明确说明

### 12.8 参考文献
- 书籍：`[编号]作者.书名.出版地:出版社,年份.` 期刊：`作者.篇名.刊名,卷(期):页码,年份.`
- 引用处用[1][2]标注，与文献表一一对应

### 12.9 附录
- 完整可运行源程序+支撑材料文件列表+AI工具使用声明(2025新规)
- 较大中间结果表格；无程序需说明

## 十三、格式规范红线

- A4纸，上下左右页边距≥2.5cm，左侧装订
- 第1页承诺书，第2页编号页，第3页摘要页
- 页码从摘要页开始，页脚中部，阿拉伯数字
- 正文≤20页，全文匿名(无学校/姓名/赛区)，电子版单PDF≤20MB
- 公式可编辑对象；图表清晰坐标轴有标签单位；表格三线表

## 十四、常见死法汇总

| ❌ 死法 | 后果 |
|---------|------|
| 摘要只写"建立了模型得到结果" | 第一印象分极低 |
| 假设堆砌万能假设(数据真实可靠等) | 评审质疑模型严谨性 |
| 公式是截图非可编辑 | 格式分扣除 |
| 图表贴上去不引用 | 无法通过格式检查 |
| 求解跳步只有结果 | 正确性无法验证 |
| 灵敏度分析只有一句"模型稳定" | 检验不充分 |
| 参考文献缺作者/页码 | 学术规范扣分 |
| 附录无源码或代码不运行 | 可能取消资格 |
| 正文出现身份信息 | 匿名性违规 |
| 超20页或缺必要章节 | 直接降档 |
