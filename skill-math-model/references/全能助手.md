## 二、问题类型识别与模型选择（三步决策法）

### 第一步：确定问题目标类型

| 目标类型 | 关键词识别 | 首选模型 | Toolkit 函数 |
|----------|-----------|---------|-------------|
| **优化** | "最优"、"最大化"、"最小化"、"最短路径" | 线性/整数规划、DP、图论 | `knapsack_01`, `bfs_path`, `dijkstra` |
| **预测** | "预测"、"趋势"、"未来"、"估计" | ARIMA、回归、GM(1,1) | `linear_regression`, `gm11_predict` |
| **评价** | "排名"、"评估"、"选择方案"、"打分" | TOPSIS+熵权法 | `topsis`, `entropy_weight`, `ahp_weight` |
| **分类** | "分类"、"识别"、"判断" | SVM、随机森林、K-means | `kmeans` |
| **随机模拟** | "随机"、"概率"、"不确定性" | 蒙特卡洛、马尔可夫链 | `monte_carlo_simulate`, `MarkovChain` |
| **博弈** | "博弈"、"竞争"、"多方"、"策略" | 纳什均衡 | `nash_equilibrium_2x2` |

### 第二步：审视数据特征

| 数据特征 | 策略 |
|----------|------|
| 样本 < 100 | 不用神经网络！用回归/SVM/灰色预测 |
| 样本 100-10000 | ARIMA、XGBoost、岭回归 |
| 样本 > 10000 | LSTM、深度学习 |
| 线性关系 | 线性回归、线性规划 |
| 非线性 | 多项式回归、核方法、神经网络 |
| 时序周期性 | ARIMA/指数平滑（别用线性回归！） |
| 共线性严重(VIF>10) | 岭回归/LASSO |
| 数据缺失贫信息 | 灰色预测GM(1,1) |
| 数据完整充足 | TOPSIS+熵权法 |

### 第三步：交叉验证诊断

```
训练误差高 + 测试误差高 → 欠拟合 → 换更复杂模型
训练误差低 + 测试误差高 → 过拟合 → 加正则化/增数据
两者都低且接近        → 模型合适 ✓
```

### 经典错误搭配（必须避免！）

| ❌ 错误 | ✅ 正确 |
|--------|--------|
| 线性回归拟合指数增长 | 多项式/Logistic/灰色预测 |
| GM(1,1)预测剧烈波动 | ARIMA（≥10组数据） |
| AHP的CR>0.1仍使用 | 熵权法修正 + TOPSIS |
| 样本<100用神经网络 | SVM/随机森林 + L2正则 |
| 评价不检验信效度 | Cronbach alpha + KMO |
| 所有问题用同一方法 | 按问题类型选择 |

## 三、论文写作规范（强制执行）

### 3.1 8段式结构

```
1. 摘要（最关键！≤1页，必须含数值结果）
2. 问题重述（用自己的话，严禁照抄原题）
3. 问题分析（附技术路线流程图）
4. 模型假设与符号说明（三线表，5-6条假设）
5. 模型的建立与求解（核心，≥50%篇幅）
6. 结果分析与检验（灵敏度+模型对比+误差）
7. 模型评价与推广（3优点+诚实缺点+推广）
8. 参考文献（≥5篇，格式统一）
附录: 完整可运行源代码
```

### 3.2 公式规范

- **重要公式单独占行、居中、带编号**（右对齐）
- 每个公式后逐一解释变量含义
- 变量用斜体，单位/函数名用正体，DOCX 中禁止 `\( \)` `$$ $$` `\\alpha`
- DOCX 下标用 `font.subscript = True`，上标用 `font.superscript = True`
- 禁止截图公式

### 3.3 图表铁律

| 规则 | 说明 |
|------|------|
| 表上图下 | 表标题在上方⚡图标题在下方 |
| 三线表 | 顶线粗 + 栏目线细 + 底线粗 |
| 独立编号 | 图1/图2/表1/表2 按序 |
| 图表解释 | 每个图表下方必须有文字结论 |
| 分辨率 | ≥200dpi，中文用中文字体 |

### 3.4 摘要三段式

```
[起始段] 背景+总括（2-3行）
[主体段] 每问一段：问题→方法→模型→结果数值
[结尾段] 特色总结+灵敏度+推广价值
关键词: xxx; xxx; xxx; xxx（4-6个）
```

### 3.5 学术句式

| 场景 | 句式 |
|------|------|
| 引出 | 「本文针对……问题，建立了……模型」 |
| 方法 | 「采用……算法对上述模型进行求解」 |
| 结果 | 「计算结果表明，最优值为……」 |
| 对比 | 「通过对比分析可知……」 |
| 局限 | 「需要指出的是，本模型在……方面尚待改进」 |

## 五、结果验证规范

### 5.1 灵敏度分析（必做）

- 关键参数 ±20% 变化，观察结果波动
- 弹性系数 = 结果变化% / 参数变化%
- 弹性<1 → 模型稳健；弹性>1 → 参数敏感

### 5.2 模型对比（必做）

- 至少与 2 种其他方法横向对比
- 对比维度：精度/速度/可解释性/鲁棒性
- 说明本文方法的适用条件和优势

### 5.3 误差分析

- 说明误差来源（模型假设/数据精度/算法近似）
- 区分系统误差和随机误差
- 给出置信区间或误差范围

## 七、常见建模问题模板

### 7.1 路径优化类
```
1. 图论化简：BFS/Floyd 计算关键节点距离
2. 决策变量：路径序列、停留天数、补给量
3. 约束条件：负重/时间/资源非负
4. 目标函数：最大化终点资金/最小化总成本
5. 求解：枚举搜索 或 DP 或 启发式算法
```

### 7.2 预测类
```
1. 数据预处理：缺失值填补、异常值检测、平稳性检验
2. 特征工程：滞后特征、滚动统计、时间特征
3. 模型选择：ARIMA(时序)/回归(多变量)/GM(1,1)(小样本)
4. 参数估计：最小二乘/最大似然
5. 验证：留出法 + 残差分析 + R^2/RMSE
```

### 7.3 评价类
```
1. 指标体系构建：目标层→准则层→指标层
2. 数据标准化：min-max/向量规范化/Z-score
3. 权重确定：熵权法(客观)/AHP(主观)/组合赋权
4. 综合评价：TOPSIS(首选)/模糊评价/灰色关联
5. 灵敏度：权重±20%观察排序变化
```

### 7.4 随机模拟类
```
1. 随机变量建模：确定分布类型和参数
2. 抽样方法：蒙特卡洛/拉丁超立方(降方差)
3. 模拟执行：N次独立模拟(≥500次)
4. 统计输出：均值/方差/置信区间/成功率
5. 收敛性：检查均值随N的变化趋势
```
