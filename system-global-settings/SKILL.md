# 系统全局设置面板

## 一、启动配置

管控开机扫描、舆情采集与缓存时效。

- **开机本地绘图软件自动扫描开关**：二元可控选项，默认开启。开启时程序初始化阶段遍历本地磁盘，完成Photoshop、Krita、GIMP三类绘图软件安装目录检索。关闭则跳过硬件环境检测，缩减程序启动耗时。
- **开机全网用户舆情自动采集开关**：二元可控选项，默认开启。开启状态下程序启动阶段合规访问技术社区、创作类论坛，采集大模型、Agent工具真实用户评价，作为评分机制动态调分数据源。关闭后系统终止舆情采集，模型分数仅依靠月度行业标准数据集基线迭代更新。
- **舆情/软件检测缓存生命周期自定义配置**：系统默认缓存时效24小时，支持自定义数值配置。缓存周期内重复启动程序，系统直接读取本地持久化缓存文件，规避重复磁盘扫描、重复外网请求，降低网络带宽损耗与Token资源占用。

## 二、模组隔离

控制拓展包数据互通、故障隔离机制。

- **模组默认数据隔离开关**：全局顶层控制开关，系统默认开启。开启状态下剧本拓展、生图拓展执行物理数据隔离，无用户显式联动指令时，两套模组无法互相读取素材库、任务参数、生成缓存。关闭则两套模组默认双向全量开放数据通道。
- **联动任务完成自动断连开关**：系统默认开启。剧本-绘图联动任务完整执行完毕后，系统即时销毁跨模组数据传输接口，恢复素材隔离状态，杜绝跨模块残留参数引发人设错乱、画面参数冲突。手动关闭后联动通道持续保持互通状态。
- **单模组故障隔离总控开关**：系统默认开启。剧本、生图任一模组出现报错、Token超限、循环死锁、接口崩溃等异常时，系统对故障模组执行运行隔离，底层核心功能、另一业务模组不受影响，可独立正常调度。手动关闭后单一模组异常将直接导致整套程序终止运行。

## 三、RAG配置

设置图文素材分库存储、文本自动压缩。

- **剧本/图像素材分区隔离存储开关**：系统默认开启。开启后向量数据库划分为两组独立向量索引空间，剧本文稿、人物设定、分镜文本归入剧本专属分区；参考图、三视图、绘图提示词、配色参数归入图像专属分区，两类素材索引完全隔离，检索无交叉干扰。关闭后全量素材混存，易产生语义检索混杂问题。
- **检索文本自动摘要压缩开关**：系统默认开启。知识库、外网检索返回超长文本时，自动抽取核心语义生成轻量化摘要送入模型上下文，抑制大模型幻觉生成、缩减检索链路Token开销。关闭后检索原始完整文本直接注入模型输入窗口。

## 四、评分阈值

自定义模型合格分数线，联动兜底流程。

- **综合评分合格阈值自定义项**：系统预设基准分数线88分，支持用户自定义数值修改。全系统所有大模型、全业务任务统一采用该阈值作为判定标准。
- 模型综合得分低于阈值时，底层核心技能包自动触发低分兜底处置流程。

## 规则联动

- 启动配置(一) → 驱动 skill-system-core 开机自检(1)与舆情采集
- 模组隔离(二) → 约束 skill-script-creation 与 skill-image-generation 数据互通边界
- 联动物断连(二.2) → 控制跨模组通道生命周期
- 故障隔离(二.3) → 保障核心系统在单模组异常时持续运行
- RAG配置(三) → 联动 skill-search-opt 检索策略与 skill-doc-parse 素材分流
- 评分阈值(四) → 触发 skill-model-scoring 低分兜底处置流程
- 全模组运行逻辑 → 强制受本面板开关约束
