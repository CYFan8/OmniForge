# 自主学习拓展包（自适应扩容核心模组）

> 元技能 — 学习其他技能并优化其输出。常驻基础 5 Skills + 自适应扩容 + 兜底管控 3 Skills

---

## 一、常驻基础Skill（永久内置，不随拓展增减变化）

### 1. 全模块素材自动归集Skill
扫描本地所有模块产出文件：剧本文档、生图图集、Blender工程、渲染素材、配置参数。自动分类切片、向量化入库RAG知识库。每新增一款拓展包，自动拓展归集文件类型白名单。

- 扫描路径: D:\ai_skill_lib\ 各模块产出目录
- 切片粒度: 剧本按场次/生图按分镜/建模按资产
- 入库格式: chunk_id → embedding → 分区索引

### 2. 合规公开素材检索Skill
仅爬取免费行业教程、开源案例（Poly Haven/Blender Demo Files/公开论文）。设置单轮检索Token上限，防止检索消耗挤占创作资源。新增拓展包自动匹配对应领域检索关键词。

- 检索范围: 开源社区/技术文档/公开教程
- Token限额: 单轮 ≤ 2000 tokens（基础）→ 阶梯扩容
- 新增拓展包: 自动注册行业标签（如Blender→#建模 #三渲二 #拓扑）

### 3. 全链路缺陷复盘记录Skill
捕捉生图崩坏、建模透视错误、剧本逻辑漏洞、三渲二光影失真，生成标准化缺陷样本库作为学习素材。每条缺陷记录含：发生时间、所属模块、缺陷类型、严重程度、上下文快照。

- 缺陷分类: 逻辑/视觉/结构/性能
- 严重等级: P0（阻断）→ P3（微瑕）
- 关联模块: 剧本/生图/建模/评分

### 4. 用户反馈采集迭代Skill
开机同步爬取GitHub Issues、技术论坛用户评价，动态调整裸模型评分、各技能权重。新增拓展包自动新增对应反馈分类（如Blender拓展→新增#建模质量 #渲染速度反馈类目）。

- 采集源: GitHub Issues / Discussions / 技术博客
- 更新频率: 每24h增量拉取
- 权重影响: 正面反馈+0.3~1分/负面反馈-0.5~2分（7日滚动均值平滑）

### 5. 学习成果全局同步Skill
完成专项学习后，自动更新对应拓展包的提示词模板、参数阈值、知识库素材，全链路同步优化输出效果。跨模块传递：生图优化参数→同步至Blender渲染参考→同步至剧本视觉描述模板。

- 同步目标: 提示词模板 / 参数阈值 / RAG素材 / 避坑清单
- 触发条件: 缺陷复盘≥5条同类 / 用户反馈≥3条同方向
- 同步范围: 目标模块 + 管线上下游联动模块

## 二、自适应动态扩容机制（核心特性）

系统实时检测已安装业务拓展包列表。每新增一款拓展包，自动追加专属学习子Skill、扩充独立知识库类目，无需手动修改配置。

### 当前已匹配 4 组拓展包专属学习子Skill

#### 匹配 Pack 1 (剧本): 
- 剧本叙事逻辑学习Skill — 分析短剧节奏、冲突密度、人物弧光
- 人物塑造学习Skill — 角色成长曲线、对白风格匹配
- 短剧分镜文案学习Skill — 镜头语言转文字模板
- 世界观搭建学习Skill — 设定一致性校验规则

#### 匹配 Pack 2 (生图):
- 人体/动物写实解剖学习Skill — 比例/肌肉/关节结构知识库
- 光影材质学习Skill — PBR物理渲染原理、金属/布料/SSS材质表现
- 三视图结构还原学习Skill — 单图→多视图空间推理
- 建模前置生图逻辑学习Skill — 锁定3D结构下的风格化叠加

#### 匹配 Pack 3 (Blender建模):
- 硬表面建模学习Skill — 布尔运算/倒角/拓扑优化
- 体块搭建学习Skill — 三视图对齐/比例锁定/低模→高模流程
- 三渲二分层渲染学习Skill — NPR着色器/线稿提取/二分色
- 镜头透视控制学习Skill — 焦距匹配/景深/安全框构图

#### 匹配 Pack 4 (自身):
- 素材分类算法学习Skill — 多模态文件自动归类
- Token限流策略学习Skill — 检索消耗与创作产出平衡
- 跨模块数据互通规则学习Skill — MCP协议兼容性
- 学习效率自评学习Skill — 学习投入产出比跟踪

### 扩容规则（后续新增拓展包自动生效）

若新增音频配音、动画分镜等拓展包：
1. 自主学习包自动生成该模块专属学习子Skill
2. 新增对应行业素材检索标签、知识库分类
3. 新增专项缺陷捕捉、复盘逻辑
4. 学习完成后自动向新拓展包下发优化参数、模板

## 三、资源兜底管控Skill

### 1. 学习任务Token阶梯限流Skill
新增大量学习类目时，平缓上调本模块总Token配额，隔离检索消耗，不挤占创作类拓展包资源。限流逻辑：基础配额4000→每新增1拓展包+800→上限12000 tokens/轮。

### 2. 学习任务优先级调度Skill
用户当前正在使用的拓展包（检测到活跃MCP会话），对应学习任务优先占用算力；闲置拓展包学习任务降低资源分配。优先级队列：活跃模块 > 近期使用(7d) > 历史资产 > 纯探索。

### 3. 学习知识库隔离Skill
不同拓展包对应的学习素材、训练参数分库独立存储，避免跨领域知识混淆引发输出错误。知识库分区：kb_script/ kb_image/ kb_blender/ kb_meta/，各分区独立embedding索引。

- **前置**: 无强制依赖，与所有拓展包松耦合
- **联动**: 读取各拓展包SKILL.md注册学习子Skill；写入RAG知识库分区；推送优化参数
- **内置**: 素材切片/向量化/检索复用skill-system-core RAG底层逻辑
