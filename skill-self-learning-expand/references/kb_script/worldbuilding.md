# 世界观搭建

> 来源: skill-script-creation 世界观归档 + chinese_script_guide.md

## 世界观归档五要素

### 1. 时代背景
- 具体年代/朝代: 提供明确时间锚点
- 科技水平: 可用的交通工具/通讯方式/武器
- 社会制度: 政治结构/阶级划分/法律体系
- 重大事件: 影响故事的宏观历史事件

### 2. 地理空间
- 核心场景清单: 每个场景标注(名称/类型/所有者/氛围)
- 场景流转图: 人物在不同场景之间的移动逻辑
- 距离与时间: 场景之间的移动耗时(避免瞬移)

### 3. 规则体系 (奇幻/科幻必须)
- 魔法/科技规则: 能做什么, 不能做什么, 代价是什么
- 规则一致性: 全文规则不变, 不能前期"魔法需咒语"后期"心念一动即可"
- 规则的代价: 每次使用规则需付出对应代价(避免角色无限强大)

### 4. 社会生态
- 经济系统: 货币/物价/主要产业
- 文化习俗: 节日/禁忌/礼仪
- 语言特色: 独有词汇/称呼方式
- 权力结构: 谁掌权/谁被压迫/权力更迭规则

### 5. 人物网络
- 势力分布: 各阵营的目标/资源/关系
- 信息流动: 谁知道什么, 信息如何传递
- 权力博弈: 阵营间的竞争/合作/背叛

## 设定一致性校验规则

### 校验维度
| 维度 | 校验内容 | 触发频率 |
|------|---------|---------|
| 时间一致性 | 事件先后顺序, 人物年龄推算 | 每场 |
| 空间一致性 | 场景间移动时间, 人物同时出现可能性 | 每场 |
| 规则一致性 | 魔法/科技规则使用是否前后一致 | 每次使用 |
| 称谓一致性 | 同一角色名/称号在全文中是否统一 | 每出现 |
| 道具一致性 | 道具特性/归属/出现时机 | 每出现 |

### 校验算法
```
1. 全文扫描 -> 提取所有设定项(时间/空间/规则/称谓/道具)
2. 每条设定项 -> 检索全文所有出现位置
3. 比对: 同一设定项在不同位置的描述是否矛盾
4. 矛盾 -> 定位冲突段落 -> 标注建议修改方案
```

## 世界观深度标准
- 短剧(1-5分钟): 1-2条核心规则, 3-5个场景, 5个以内主要人物
- 剧集(10-30分钟): 3-5条核心规则, 8-15个场景, 5-10个主要人物
- 电影(90-120分钟): 完整的五要素体系, 10+场景, 完整的势力分布图
- 长篇/系列: 全套世界观 + 年代表 + 多势力博弈图谱

## 常见世界观缺陷
- 规则自相矛盾: 前期设定了"只有贵族能用魔法", 后期平民角色随意施法
- 地理混乱: 角色"向东走三天"却到达了之前"在西边"的地点
- 经济崩溃: 角色的财富/资源来源无法支撑其行为
- 信息全知: 不同阵营的角色共享不应知道的情报
- 时代错位: 设定在古代却出现现代概念/词汇

## 世界观文档输出格式
```
【项目名称】XXX
【时代】XXX年代, XXX社会制度
【核心规则】1. ... 2. ... 3. ...
【场景清单】
  - 场景A: 类型(室内/室外), 功能, 关联角色
  - 场景B: ...
【人物势力】
  - 阵营A: 目标/资源/关键人物
  - 阵营B: ...
【时间线】
  - T0: 起始事件
  - T1: 转折事件
  - T2: 高潮事件
  - T3: 结局
```
