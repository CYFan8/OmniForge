# 学习效率自评

> 来源: skill-self-learning-expand

## 学习投入产出比跟踪

### 评估维度
| 维度 | 指标 | 目标值 |
|------|------|--------|
| 素材覆盖率 | 已索引素材/全模块总素材 | >90% |
| 检索命中率 | 有效检索/总检索次数 | >70% |
| 缺陷发现率 | 捕获缺陷/实际缺陷(估算) | >60% |
| 学习转化率 | 已同步优化/已发现缺陷 | >80% |
| Token效率 | 学习产出(改善项)/学习Token消耗 | >0.5/Ktokens |

### 学习质量评估
- 正面信号: 缺陷重复率下降 / 用户手动修正次数减少 / 同类问题3次内自动解决
- 负面信号: Token消耗增长但改善停滞 / 检索结果噪音增多 / 跨模块同步冲突

### 触发重新学习的条件
1. 缺陷重复出现 >= 3次 -> 标记学习失败 -> 重新检索+更新知识库
2. 用户手动覆盖 >= 2次 -> 学习成果标记为"用户不认可" -> 回滚
3. 素材新增 >= 20% -> 触发增量学习

### 学习健康度分数
```
健康度 = (素材覆盖率 x 0.3 + 命中率 x 0.2 + 转化率 x 0.3 + 效率 x 0.2)
>85: 优秀, 保持
70-85: 正常, 继续
<70: 需干预, 检查检索策略/知识库污染
```
