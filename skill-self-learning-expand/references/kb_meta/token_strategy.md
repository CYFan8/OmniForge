# Token限流策略

> 来源: skill-system-core 弹性Token管控 + 自主学习兜底管控

## 配额分配模型
```
总Token预算 = 基础配额 + 拓展加成
基础配额 = 35000 (固定)
拓展加成 = 已安装拓展包数 x 4000
全局上限 = 50000 (硬封顶)

当前 (4拓展包): 35000 + 4x4000 = 51000 -> 压到50000
```

## 检索消耗隔离
| 优先级 | 消耗类型 | 预算占比 | 超限行为 |
|--------|---------|---------|---------|
| P0 | 创作核心(剧本/生图/建模) | 70% | 不可抢占 |
| P1 | 学习检索(公开素材) | 20% | 阶梯限流 |
| P2 | 舆情采集/日志 | 10% | 首先被压缩 |

## 阶梯限流逻辑
- Token消耗 < 80%: 全部功能正常
- Token消耗 80-95%: 精简检索文本, 关闭外网舆情采集
- Token消耗 > 95%: 关闭全部外网检索, 仅保留本地RAG
- 触及全局封顶: 仅核心业务(剧本+生图+建模)运行

## 学习任务Token配额
- 基础: 4000 tokens/轮
- 每新增1拓展包: +800 tokens
- 上限: 12000 tokens/轮
- 学习任务独占配额, 不与创作任务共享
