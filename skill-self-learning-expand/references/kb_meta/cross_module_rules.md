# 跨模块数据互通规则

> 来源: system-global-settings + 全部拓展包联动接口

## 默认状态: 模块隔离
- 剧本/生图/建模三拓展包默认物理隔离
- 无用户联动指令时, RAG分区独立, 缓存独立, 日志独立
- 隔离由 system-config.yaml 2.1节控制

## 联动触发条件
1. 用户明确下发联动指令 (如"把剧本分镜推送到生图模块")
2. 管线任务自动检测上下游依赖 (如Blender建模检测到三视图参考图缺失)
3. 自主学习包跨模块学习同步

## MCP数据互通协议
- 文本数据: 通过MCP通道传递结构化JSON
- 图像数据: 传递本地文件路径, 不传原始二进制进上下文
- 3D数据: 传递.blend文件路径, 通过Blender bpy桥接读取
- 大文件(>50MB): 仅传路径+元数据摘要

## 通道生命周期
- 打开: 联动指令触发 -> 创建临时数据通道
- 传输: 单向推送 (剧本->生图, 生图->建模, 建模->生图)
- 关闭: 任务完毕 -> 即时销毁通道 -> 恢复隔离
- 异常关闭: 任务超时/崩溃 -> 自动断连 -> 记录日志

## 跨模块冲突检测
- 版本不匹配: 拓展包版本号不一致 -> 弹窗告警
- 资源抢占: 两模块同时请求同一GPU -> 按优先级排队
- 死锁检测: 循环依赖(如生图<->建模双向等待) -> 超时断连
