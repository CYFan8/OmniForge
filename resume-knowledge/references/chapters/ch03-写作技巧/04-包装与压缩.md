# ch03-04 经历的包装与压缩

> 包装 ≠ 造假。包装 = 把真实经历用最好的方式讲出来。

---

## 包装 vs 造假

| 包装（✅） | 造假（❌） |
|-----------|-----------|
| "整理文件"→"建立归档体系" | 没做过的事硬说做过 |
| 用数据呈现真实成果 | 虚构数据 |
| 突出最有价值的部分 | 编造整个项目 |

---

## 压缩技巧

### 删冗余词
```
❌ 在实习期间，我主要负责了团队的文件管理和归档工作
✅ 建立文件归档体系，团队查找效率提升 80%
```

### 证书简写
```
CFA Level 2 Passed (2025.06)
```

### 项目描述结构
```
项目名称 — 一句话摘要
· bullet 1（成果+数据）
· bullet 2（核心行动）
· bullet 3（技术/工具）
```

---

## 每段经历 2 行原则

每条 bullet 不超过 2 行。超过 = 太啰嗦 = 要拆成两条或删冗余。
