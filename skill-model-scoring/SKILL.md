# 通用模型评分机制（5项）

## 1. 裸模型评测

单独检测大模型原生基础能力。评测阶段不加载任何技能包、第三方插件、辅助工具。固定值域40~100分，40分为行业最低兜底分值，100分为同期同类模型性能上限。五大核心评测维度：长文本语义理解、逻辑推导能力、原创文本创作、标准化提示词生成、结构化文档解析。权重占综合总分45%。双数据源分数迭代体系：月度全局基线校准（批量爬取全球通用专业评测数据集，Min-Max归一化统一标尺）+ 每日动态微调（抓取全网创作者、技术从业者真实使用评价，实时调整各维度得分）。

## 2. 综合打分

结合底层工具调度计算整体得分。第二层全技能搭载综合分：Agent六大通用评测维度（RAG向量库调度稳定性、多工具循环校验管控、第三方软件MCP桥接兼容性、图像像素比对执行精度、批量素材自动化处理效率、绘图软件分发交互完整性）。权重占综合总分55%。全局统一固定权重配比45%:55%，无差异化倾斜。六大核心刚需维度（文档解析、RAG检索、图像解析、软件桥接、像素循环校验、人设一致性）任一低于78分→模型最终综合总分统一扣除5~10分，规避核心功能失效但综合分数虚高的失真问题。合格判定标准：以全局设置面板用户自定义综合分数线为唯一判定阈值（默认88分）。

## 3. 分层补强

根据低分自动补齐缺失工具组件。Agent刚需维度得分划分三层补强策略：低于78分时，系统自动全量下载安装对应配套插件、绘图软件，完整补齐缺失功能；78~88分时，仅加载精简核心功能组件，平衡功能完整性与算力、Token资源消耗；88~95分时，仅保留基础轻量化校准工具，最大化节约系统资源。裸模型原生短板仅加载轻量化辅助组件（长文本摘要工具、文案修正插件），大模型底层算法原生缺陷无法通过外部工具彻底根除。

## 4. 低分兜底

分数不足时切换轻量化运行模式。系统自动完成全套插件、绘图工具下载安装补强，补强完成后重新核算模型综合得分。若最终得分仍未达到全局设置自定义合格阈值，自动生成结构化可视化诊断报告，完整标注未达标刚需维度、系统全部执行的补强操作明细。向用户提供两套自主可选处置方案：方案一轻量化降级运行模式（系统自动关闭高精度像素校验、批量高清图像生成等高资源消耗功能，仅保留剧本撰写、简易单图生成基础功能可用）；方案二智能适配模型推荐（调取全网裸模型评分总库，筛选裸模型基础分≥82分的适配大模型展示，支持一键切换，原有RAG向量库全量素材、用户全局配置完整复用，无需重复导入素材）。

## 5. 配套资源联动约束

评分机制与底层系统的资源联动规则。每日开机采集的全网用户舆情、模型全维度评分计算记录，全部自动写入底层模组异常日志（通过skill-system-core日志系统持久留存），支持用户导出完整记录用于参考。评分机制全部外网基线校准请求、舆情采集请求，统一复用底层外网自适应限流与重试容错机制，确保无无节制高频网络请求。调分约束清洗规则：自动剔除水军灌水、无依据情绪化吐槽、无关闲聊文本，仅保留长文本理解、逻辑推理、文本创作、文档解析、提示词生成五大维度有效评价参与计分。量化加减分标准：正向有效用户反馈对应能力维度单次加分0.3~1分，单日单维度加分上限3分；逻辑崩坏、长文本截断、提示词生成失效等严重缺陷对应维度单次扣分0.5~2分，大规模集中投诉单日最高扣分8分。分数平滑约束：单日单模型总分浮动区间±5分，叠加7日滚动均值平滑处理，规避单条极端评价造成分数大幅波动。

## 依赖清单

### 数据采集与评测
| 组件 | 用途 | 安装命令 |
|------|------|---------|
| requests + beautifulsoup4 | 月度评测数据集爬取 | `pip install requests beautifulsoup4 lxml` |
| scrapy | 大规模舆情采集框架 | `pip install scrapy` |
| pandas + numpy | 评分数据统计与归一化 | `pip install pandas numpy scikit-learn` |

### 评分存储与可视化
| 组件 | 用途 | 安装命令 |
|------|------|---------|
| matplotlib + seaborn | 评分趋势可视化 | `pip install matplotlib seaborn` |
| pyyaml | 评分配置与结果持久化 | `pip install pyyaml` |

### 联动依赖
- 舆情采集请求 → 复用 skill-system-core 外网限流机制（requests库）
- 评分计算日志 → 写入 skill-system-core 日志系统
- 模型切换触发 → 调用 skill-system-core 模型切换模块

---

## 规则联动

- 合格阈值 → 受 system-global-settings 评分阈值配置
- 裸模型评测(1) → 调用 skill-system-core 开机自检舆情采集数据
- 综合打分(2) → 联动 skill-system-core Agent调度与工具链
- 分层补强(3) → 联动 skill-system-core 低分兜底与工具更新
- 低分兜底(4) → 联动 skill-system-core 模型切换
- 配套约束(5) → 复用 skill-system-core 日志系统与外网限流
- 评分日志 → 写入 skill-system-core 日志系统
