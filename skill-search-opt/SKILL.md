# 精准检索增强策略

## 核心原则

`
边界优先 → 命中即停 → 效率优先 → 最小空间
`

- **边界优先**：先确定检索范围边界，缩小搜索域
- **命中即停**：找到目标信息后立即停止，不做无意义的全面扫描
- **效率优先**：每次查询都应当是必要的，无冗余检索
- **最小空间**：检索过程占用最小上下文空间

## 二、查询准入规则

### 何时使用 webFetch

`
信息页面评分 ≥ 7/10 → webFetch 直接获取
信息页面评分 < 7/10 → 先用文本版/摘要确认价值
`

**评分维度**：
- 相关性（0-4）：与查询目标的相关程度
- 权威性（0-3）：来源的可信度
- 时效性（0-3）：信息的更新程度

总分 ≥ 7 才值得完整 webFetch。

### 查询构造原则

1. 一次只查一个明确问题（非「了解XX」这种模糊查询）
2. 查询词尽量控制在 3-6 个关键词内
3. 优先使用 Site:domain 限定范围
4. 优先查官方文档/原始来源，而非二手解读

## 四、结果评估

命中后对结果做快速评估：

| 标准 | 通过条件 |
|------|---------|
| 完整性 | 回答核心问题，无明显信息缺口 |
| 准确性 | 与已知事实无矛盾，来源权威 |
| 时效性 | 信息在合理时效范围内 |
| 可验证 | 提供了可查证的来源或数据 |

三项以上通过 → 判定为有效命中，交付结果。
少于三项 → 继续搜索优化（L2 或 L3）。
